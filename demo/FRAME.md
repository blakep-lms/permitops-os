# FRAME.md — PermitOps OS V3 Twitter / Nous Submission Video

## Objective
Make PermitOps OS feel like a serious, safe business-operations product — not a generic AI demo — while fixing V1/V2 failures: crowded text, robotic narration, weak music, and unclear hierarchy.

## Audience + channel
- Audience: Nous hackathon reviewers, X/Twitter viewers, service-business operators.
- Channel: X demo post + Discord/news/research submission context.
- Duration: ~94s master.
- Aspect: 16:9, 1920×1080.

## Narrative
Hook → operator reveal → intake proof → packet readiness → approval/safety → Stripe earn/spend gates → vertical expansion → CTA.

## Visual language
- Mood: official, calm, operator-grade.
- Palette: deep navy, permit blue, success green, amber review, paper white.
- Text: sparse, scene-level captions only; no paragraph captions.
- UI: synthetic command-center cards, receipts, approval gates, pipeline rails.

## Audio
- Voice: Edge TTS, sped gently to remove drag while preserving clarity.
- Music: custom warm operations bed, low/subtle.
- SFX: soft UI accents mixed low at scene changes.

## Truth boundary
Clean-room demo with synthetic/redacted data. Real workflow pattern. Approval-gated execution. No autonomous filing/sending/spend claims.
